verifyAttribute()

The verifyAttribute() method maps directly to getAttribute() method on WebdriverIO.

The attribute of the HTML element you're looking for will always be the assert key in your "asserts" object. Consider the following element (any HTML element will work, we chose an input): 

<input class="test" id="input1" required type="text" data-type="test" />

if you wanted to verify that the input field had the type of text, you could do the following: 

{
  "selector": "#input1",
  "verify": [
    {
      "method": "verifyAttribute",
      "asserts": {
        "type": "text"
      }
    }
  ]
}

Let's say that we wanted to make sure the disabled attribute doesn't exist on our input. We could make sure that was the case with: 

{
  "selector": "#input1",
  "verify": [
    {
      "method": "verifyAttribute",
      "asserts": {
        "disabled": null
      }
    }
  ]
}

We can also test for boolean HTML attributes, so I can make sure that input has a required attribute on it. 

{
  "selector": "#input1",
  "verify": [
    {
      "method": "verifyAttribute",
      "asserts": {
        "required": true
      }
    }
  ]
}

In fact, here is a test that we have for the input element on this page.

'use strict';


var drive = require('datadriver');


drive.verify({
  "steps": [
    {
      "execute": [
        {
          "action": "redirect",
          "page": "api/verify/verifyAttribute.html"
        }
      ]
    },
    {
      "selector": "#input1",
      "verify": [
        {
          "method": "verifyAttribute",
          "asserts": {
            "disabled": null,
            "required": true,
            "data-type": "test",
            "class": "test",
            "type": "text"
          }
        }
      ]
    }
  ]
});

And this is our expected output:

$ grunt datadriver:verify --url=http://datadriver.io

✓ the "type" attribute of "#input1" should equal "text"

Call verifyAttribute() directly

You can also call verifyAttribute() directly in your tests for even more granularity. The method takes an object with four keys. For example, this test is verifying the class attribute on the #input1 element (visible on this page). 

'use strict';


var drive = require('datadriver');


drive.verifyAttribute({
  selector: '#input1',
  assertKey: 'class',
  assert: 'test',
  mode: [ 'equal' ]
});

The mode key is an array because we can pass multiple ChaiJS modes to our test. Modes should be read with a prefix of should, since we're using the should interface from ChaiJS. For example, if we wanted to assert that the class should not equal to "error", then we could do this: 

'use strict';


var drive = require('datadriver');


drive.verifyAttribute({
  selector: '#input1',
  assertKey: 'class',
  assert: 'error',
  mode: [ 'not', 'equal' ]
});

We can also set the assert mode in our data structures.
