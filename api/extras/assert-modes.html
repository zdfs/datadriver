Assert Modes

Most of the time, the default assertion mode of equal is going to be all you need for a decent test. But there are many more use-cases where a strict equal assertion isn't going to work. Maybe you're testing for a value that needs to be above 2. How do you write a test for that?

Simple. You can change the assertion mode. Let's see what this might look like.

hello!

¡hola!

bonjour!

'use strict';


var drive = require('datadriver');


drive.verify({


  "steps": [
    {
      "execute": [
        {
          "action": "redirect",
          "page": "api/extras/assert-modes.html"
        }
      ]
    },
    {
      "selector": ".grid--third",
      "verify": [
        {
          "method": "verifyCount",
          "assert-mode": "be above",
          "asserts": {
            "count": 2
          }
        }
      ]
    }
  ]


});

And here is our expected output:

$ grunt webdriver:verify --url=http://datadriver.io
Running "webdriver:verify" (webdriver) task


⚙ redirecting to http://datadriver.io/api/extras/assert-modes.html
✓ the number of ".grid--third" elements on the page should to be above "2"


1 passing

Your assertion modes correspond to most of the ChaiJS API. We use the should interface, so the assertion mode in the above example reads "should be above".

We don't support the ChaiJS API perfectly, but we do support a lot where it makes sense for our declarative syntax.
